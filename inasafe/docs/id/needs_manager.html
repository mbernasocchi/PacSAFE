Pengaturan Kebutuhan Minimum¶

Dalam versi 2.2, sistem standar kebutuhan minimum telah mengalami banyak perbaikan. Pengaturan kebutuhan minimum adalah sebuah piranti (tool) baru yang memperkenankan Anda untuk menentukan ‘profil’ dari kebutuhan minimum untuk membuatnya sesuai dengan yang diterapkan di negara atau daerah Anda.

Pengaturan Kebutuhan Minimum

Kami memuat InaSAFE dengan profil standar untuk:

  • Indonesia - berdasarkan pada ‘Perka 7/2008’
  • Filipina (berdasarkan pada ‘Guidelines on Evacuation Center Coordination and Management Joint Memorandom Circular No. Series of 2013’)

Terdapat dua konsep utama yang harus diperhatikan dalam mengatur kebutuhan minimum:

  • profil
  • sumber daya

Profil Kebutuhan Minimum¶

Profil merupakan kumpulan sumber daya yang mendefinisikan kebutuhan minimum untuk sebuah negara atau daerah. Biasanya sebuah profil didasarkan pada sebuah standar regional, nasional atau internasional. Sumber daya yang didefinisikan dalam profil tergantung dari kondisi dan kebiasaan lokal dari tempat dimana rencana kontinjensi dirancang.

Sebagai contoh di timur tengah, nasi merupakan makanan pokok. Namun di Afrika Selatan, jagung merupakan makanan pokok. Oleh karena itu, rencana kontinjensi harus disesuaikan dengan kebutuhan lokal.

Sumber daya untuk kebutuhan minimum ¶

Setiap item di dalam profil kebutuhan minimum merupakan sebuah sumber daya. Setiap sumber daya dideskripsikan sebagai sebuah kalimat sederhana seperti:

“Setiap individu harus disediakan 2,8 kg beras setiap minggunya.”

Dengan meng-klik pada isian sumber daya di dalam jendela profil, dan kemudian klik pada ikon pensil hitam Anda akan dapat mengatur sumber daya dengan menggunakan editor. Alternatif lainnya, Anda dapat membuat sumber daya baru untuk sebuah profil dengan meng-klik pada tanda + hitam yang terdapat pada pengaturan profil. Anda juga dapat menghilangkan sumber daya dari sebuah profil menggunakan ikon - pada pengaturan profil.

Pengatur Sumber

Ketika pindah ke mode ubah atau tambah sumber, pengaturan kebutuhan minimum akan berubah menjadi seperti pada tampilan di atas. Setiap sumber daya dideskripsikan dengan detail:

  • nama sumber daya, contohnya: Nasi
  • penjelasan mengenai sumber daya, contohnya: Makanan pokok
  • satuan dari sumber daya tersebut, contohnya: kilogram
  • bentuk majemuk dari satuan, contohnya: kilogram
  • singkatan untuk satuan, contohnya: kg
  • alokasi default untuk sumber daya, contohnya: 2.8. Angka ini dapat diganti setiap kali analisis.
  • minimal alokasi yang dperbolehkan
  • maksimal alokasi yang diperbolehkan yang digunakan untuk mengatur batas atas yang masuk akal untuk sumber daya
  • sebuah kalimat terbaca yang digunakan untuk mengkompilasi kalimat untuk menjelaskan sumber daya di laporan.

Parameter ini mungkin semuanya sudah cukup jelas, tapi kalimat terbaca mungkin membutuhkan penjelasan lebih detil. Kalimat dikompilasi menggunakan sistem penggantian token keyword sederhana. Token-token berikut dapat digunakan:

  • {{ Default }}
  • {{ Unit }}
  • {{ Units }}
  • {{ Unit abbreviation }}
  • {{ Resource name }}
  • {{ Frequency }}
  • {{ Minimum allowed }}
  • {{ Maximum allowed }}

Ketika token diletakan di kalimat, token akan digantikan dengan nilai aslinya saat pembuatan laporan. Contoh berikut menunjukkan kalimat bertoken yang memuat semua keyword:

Setiap pengungsi harus disedakan {{ Default }} {{ Unit }}/{{ Units }}/{{ Unit abbreviation }} {{ Resource name }}. Jumlahnya tidak kurang dari {{ Minimum allowed }} dan tidak lebih dari {{ Maximum allowed }} yang harus diberikan {{ Frequency }}.

Akan menghasilkan kalimat yang mudah dibaca seperti ini:

Setiap pengungsi harus disediakan 2.8 kilogram/kilogram/kg beras. Jumlahnya tidak kurang dari 0 dan tidak lebih dari 100 yang harus diberikan per hari.

Setelah Anda mengisi bagian-bagian dari sumber daya, klik tombol Simpan sumber daya untuk kembali ke tampilan profil. Anda akan melihat bahwa sumber daya baru telah ditambahkan di daftar sumber daya dari profil tersebut.

Mengelola profil¶

Sebagai tambahan profil-profil standar di InaSAFE, Anda dapat membuat profil baru, bisa dari awal, atau berdasarkan yang sudah ada (yang dapat anda modifikasi).

Gunakan tombol Baru untuk membuat profil baru. Ketika diminta, berikan nama profil misalnya :keyboard:`JakartaProfile`.

Catatan: Profil harus disimpan di folder home Anda di .qgis2 /minimum_needs agar InaSAFE dapat mendeteksinya.

Cara lain untuk membuat profil baru adalah dengan menggunakan Simpan Sebagai untuk menduplikasi profil yang sudah ada. Profil hasi duplikasi dapat diubah sesuai kebutuhan spesifik Anda.

Profil Aktif¶

Sangat penting untuk dicatat bahwa profil manapun yang Anda pilih dari daftar Profile, akan digunakan sebagai profil aktif yang digunakan sebagai basis untuk semua analisis terkait kebutuhan minimum.
